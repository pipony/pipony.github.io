df

7/dev/vda1       41152812 21695184  17553920  56% /

fdisk

fdisk是创建和维护磁盘分区表的命令。执行用户需要root。

 2fdisk: invalid option -- '-'
 3Usage:
 4 fdisk [options] <disk>    change partition table
 5 fdisk [options] -l <disk> list partition table(s)
 6 fdisk -s <partition>      give partition size(s) in blocks
 7
 8Options:
 9 -b <size>             sector size (512, 1024, 2048 or 4096)
10 -c[=<mode>]           compatible mode: 'dos' or 'nondos' (default)
11 -h                    print this help text
12 -u[=<unit>]           display units: 'cylinders' or 'sectors' (default) # 搭配”-l”参数列表，会用分区数目取代柱面数目，来表示每个分区的起始地址
13 -v                    print program version
14 -C <number>           specify the number of cylinders # 柱面数
15 -H <number>           specify the number of heads # 磁头数
16 -S <number>           specify the number of sectors per track # 每个磁道的扇区数

理论基础：

块（block）与扇区（sector）的区别。 扇区是对硬盘而言，扇区是硬盘的最小存储单位，块是对文件系统而言，块是文件系统最小存取单位。一般而言，一个扇区大小为512B，一个块大小为4KB，一个block是由连续的8个sector组成。

https://www.cnblogs.com/joshua317/p/15407969.html

 1#参数说明
 2Device：分区名称；
 3Boot：是否是活动分区。活动分区只能是主分区，一个硬盘只能有一个活动的主分区；一个硬盘的主分区与扩展分区总和不能超过4个。硬盘分区遵循着“主分区→扩展分区→逻辑分区”的次序原则，而删除分区则与之相反。
 4 主分区：一个硬盘可以划分多个主分区，但没必要划分那么多，一个足矣。
 5 扩展分区：主分区之外的硬盘空间就是扩展分区，
 6 逻辑分区：是对扩展分区再行划分得到的。
 7Start：分区柱面的开始下标；
 8End：分区柱面的结束下标；
 9Blocks：该分区的块数量。
10Id：各种分区的文件系统不同，如有ntfs分区，fat32分区，ext3分区，swap分区等。每一种文件系统都有一个代号，对应这里的Id。常见的文件系统ID有：
11 f：FAT32 Extend,只限于扩展分区。
12 86：NTFS。
13 7：HPFS/NTFS
14 b：FAT32。
15 83：Linux Ext2。
16 82：Linux 交换区。
17System：文件系统名称。

常见用法

 1# 打印所有设备（树状形式）
 2[root@iZbp15qc4wmx335c268l5mZ ~]# lsblk -a
 3NAME   MAJ:MIN RM SIZE RO TYPE MOUNTPOINT
 4vda    253:0    0  40G  0 disk 
 5└─vda1 253:1    0  40G  0 part /
 6
 7# NAME : 这是块设备名。
 8# MAJ:MIN : 本栏显示主要和次要设备号。
 9# RM : 本栏显示设备是否可移动设备。1说明是可移动设备。
10# SIZE : 本栏列出设备的容量大小信息。例如40G表明该设备大小为40GB。
11# RO : 该项表明设备是否为只读。0表明不是只读的。
12# TYPE :本栏显示块设备是否是磁盘或磁盘上的一个分区。
13# MOUNTPOINT : 本栏指出设备挂载的挂载点。

磁盘格式化，挂载/卸载

磁盘新分好区了，使用lsblk -f查看时唯一的区别是：旧的硬盘分区都有一个UUID和挂载目录MOUNTPOINT，而新建的硬盘分区是没有的。就需要完成硬盘分区的格式化与目录挂载。所谓格式化，其实就是安装文件系统。

格式化

 1[root@iZbp15qc4wmx335c268l5mZ ~]# mkfs --help
 2Usage:
 3 mkfs [options] [-t <type>] [fs-options] <device> [<size>]
 4
 5Options:
 6 -t, --type=<type>  filesystem type; when unspecified, ext2 is used
 7     fs-options     parameters for the real filesystem builder
 8     <device>       path to the device to be used
 9     <size>         number of blocks to be used on the device
10 -V, --verbose      explain what is being done;
11                      specifying -V more than once will cause a dry-run
12 -V, --version      display version information and exit;
13                      -V as --version must be the only option
14 -h, --help         display this help text and exit
15
16For more information see mkfs(8).

1# 如把分区sdb1格式化为ext4文件系统
2mkfs -t ext4 /dev/sdb1
3# 或者是
4mkfs.ext4 /dev/sdb1
5mkfs.xfs /dev/vdb # 格式化为xfs文件系统

1# 查看支持的文件系统
2[root@iZbp15qc4wmx335c268l5mZ ~]# mkfs (输入mkfs按两下tab键)
3mkfs         mkfs.btrfs   mkfs.cramfs  mkfs.ext2    mkfs.ext3    mkfs.ext4    mkfs.minix   mkfs.xfs

目录挂载

1mkdir -p /data;        #新建一个目录用于挂载新硬盘
2mount /dev/sdb1 /data;   #挂载新硬盘到目录

完成格式化和挂载后，此时再使用lsblk -f查看分区，可以看到该分区的UUID和MOUNTPOINT都有值了。使用df -h也可以查看挂载结果。

自动挂载

上面完成的挂载关系，在操作系统重启之后就失效了。如果希望下次操作系统重启的时候自动挂载，需要在/etc/fstab文件中添加下面的一行记录实现自动挂载。

参数说明可参考： https://help.aliyun.com/document_detail/194379.htm?spm=a2c4g.11186623.0.0.15757d02qNoM4R#task-2000719

1/dev/sdb1  /data  ext4 defaults 0 0

如果是root用户，也可使用以下命令修改/etc/fstab：

参考 https://help.aliyun.com/document_detail/131111.html

1echo `blkid /dev/sdb1 | awk '{print $2}' | sed 's/\"//g'` /data ext4 defaults 0 0 >> /etc/fstab
2# 在本机上试试看echo的内容
3[root@iZbp15qc4wmx335c268l5mZ ~]# echo `blkid /dev/vda1 | awk '{print $2}' | sed 's/\"//g'` /data ext4 defaults 0 0
4UUID=9f2d3e15-a78a-4f3d-8385-0165b4b67864 /data ext4 defaults 0 0
5# 其中blkid命令是查看分区的uuid

执行如下命令会重新加载配置文件，并使其生效：

1mount -a

目录卸载

1umount /media/test  # 将挂载在/media/test 目录下的文件系统卸载
2umount -f  /media/test # 强制卸载
