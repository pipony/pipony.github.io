Linux-管道篇

概念

管道（pipe）可以连接两个或多个命令，将前一个命令的标准输出作为后一个的标准输入（这也就意味着，若某命令不能支持标准输出/输入的话，则无法应用管道命令）

格式：命令1 | 命令2 | ...

管道和重定向（命令1 > file）的区别：重定向是接收对象是文件，而管道是接命令的。

管道（又称无名管道）和命名管道FIFO的区别：等后面学到命名管道了再说

注意点

  • 管道只能传输正确输出，而无法传输错误信息。从管道接连输出的标准错误会混合到一起。

常见相关命令

grep (Global search REgular expression and Print out the line)

根据模式（正则表达式）对文本逐行扫描，找到并打印匹配的行

格式：

 1grep [options] [pattern] file
 2or
 3command | grep [options] [pattern]
 4
 5[options]
 6	-v # 反向匹配 invert (等同于--invert-match)
 7	-n # 输出加上行号
 8	-i # 忽略大小写匹配 ignorecase
 9	-c # 只输出匹配的行数
10	-E # 适用egrep匹配
11	--color=auto # 为过滤结果添加颜色 (通常在alias中默认会有这个参数，如alias grep='grep --color=auto')
12	-w # 整词对比，而不是字符
13	-o # 只输出匹配到的字符串本身（匹配到多少个就会输出几行） （常用于与wc -l配合获取到匹配的个数，而不是行数）
14    -q # 静默模式（stderr除外，不输出任何东西 quiet） 等同于：--quiet, --silent （不打印任何标准输出，如果有匹配的内容则返回状态值0，常用于if判断等）
15    -l # 只输出匹配的文件名
16    -r # 递归模式，查找所有层级子目录里的文件
17[pattern]
18	# 匹配模式，见正则表达式篇

cut

根据某条件（分隔符）分割输入的内容，并输出特定的分割结果。但是cut只支持单个字符的分隔符。

格式：

 1cut [options] file
 2or
 3command | cut [options]
 4
 5[options]
 6	-d # 指定分割符（默认分隔符是Tab，而不是空格），常与-f配合适用。分隔符只能是单个字符。
 7	-f # 指定输出分割后的第几列
 8    -c # 指定输出某区间结果（以字符为单位）
 9    -b # 以字节为单位指定某区间结果
10    –complement # 反选选中的部分，即输出除去该条件的内容
11	–output-delimiter # 修改输出时使用的分隔符。
12	--only-delimited # 不输出不包含分隔符的列。

例子：

 1[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testCut.txt  # 新建测试文件
 2i am cindy
 3
 4it.so.hot
 5butiamcold
 6但是我冷呀
 7[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testCut.txt | cut -d '.' -f 2  # 第三行有匹配到分隔符，并输出第2个内容。其他行没有匹配到，全部输出。

sort

针对文本文件的内容，以行为单位进行排序

 1sort [options] file
 2or
 3command | sort [options]
 4
 5[options]
 6	-n # 以数字格式来大小排序（默认是以字符串格式）
 7	-r # 反向排序（从大到小）
 8	-k # 以某列进行排序
 9	-t # 指定分隔符（默认分割符为空格）
10	-b # 忽略每行前面的空格再排序

例子：

 1[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testSort.txt
 212 13 14
 32 3 4
 42 1 5
 5[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testSort.txt | sort  # 按照字符串格式比较，即依次比较第1,2..个字符

13[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testSort.txt | sort -n  # 按数字格式比较，即2<12

1612 13 14
17[cindy@iZbp15qc4wmx335c268l5mZ ~]$ vi testSort.txt

22[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat testSort.txt | sort -k 2 -t ',' -n  # 分隔符为逗号，根据第2列的数字格式排序

wc

用于统计文件的字节数、字数、行数

 1wc [options] file
 2or
 3command | wc [options]
 4
 5[options]
 6	-c # 统计字节(char)数
 7	-l # 行数(line)
 8	-m # 字符数
 9	-w # 字数(word)，一个字被定义为由空白、跳格或换行字符分隔的字符串。
10	-L # 获取最长行的长度

uniq

用于检查或删除重复的列，可以与sort结合使用。

 1uniq [options] file
 2or
 3command | uniq [options]
 4
 5[options]
 6	- uniq filename (cat filename | unique) # 显示删除重复行的文件内容，但不改变原文件内容
 7	-u # 仅显示只出现一次的行(unique)，与`uniq filename`效果不同
 8	-d # 仅显示有重复的行(dupli)
 9	-c # 在每一行前面标上重复的行数(count)
10	-i # 忽略大小写(ignore)
11	-f # 忽略比较第几栏（以空格为分割符）
12# 需要注意的是，判断重复行的范围仅是相邻行。若不是相邻行的重复行，并不会被识别为重复行。若要实现将不相邻的重复行也视为重复的uniq命令，则要配合上sort命令：
13	sort filename | uniq

例子：

 1[cindy@iZbp15qc4wmx335c268l5mZ ~]$ vi testUniq.txt

tee

标准输入 -> tee -> 标准输出**+文件**，即实现既将执行结果输出，并保存到文件中。

1command | tee [options] filenameßß
2
3[options]
4	-a # 追加写到文件（默认是覆盖写）
5	-i # 忽略中断信号
6	- - ... # 重复多次
7	> dev/null # 静默输出（只输出到文件）
8
9# 注：默认会将标准输入的信息通过tee输出到文件中（即，过滤错误信息），如需要一并输出错误信息，则要结合 2>&1

例子：

 1[cindy@iZbp15qc4wmx335c268l5mZ ~]$ whoami | tee tee.txt
 2cindy  # 1、控制台输出
 3[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat tee.txt
 4cindy  # 2、文件输出
 5[cindy@iZbp15qc4wmx335c268l5mZ ~]$ whoami | tee tee.txt
 6cindy
 7[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat tee.txt  # 覆盖写
 8cindy
 9[cindy@iZbp15qc4wmx335c268l5mZ ~]$ whoami | tee -a tee.txt
10cindy
11[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat tee.txt  # 追加写
12cindy
13cindy
14[cindy@iZbp15qc4wmx335c268l5mZ ~]$ echo "hi" | tee tee1.txt tee2.txt  # 同时输出到多个文件
15hi
16[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat tee1.txt
17hi
18[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat tee2.txt
19hi
20[cindy@iZbp15qc4wmx335c268l5mZ ~]$ echo "good" | tee - -  # 额外重复输出2次
21good
22good
23good
24[cindy@iZbp15qc4wmx335c268l5mZ ~]$ echo "good" | tee good.txt > /dev/null
25[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat good.txt  # 仅输出到文件中
26good
27[cindy@iZbp15qc4wmx335c268l5mZ ~]$ ls kkk | tee out

29[cindy@iZbp15qc4wmx335c268l5mZ ~]$ cat out  # 正常没有把错误信息输出到文件中
