开始Hugo+Github Pages

Posted by cindy.H on Sun, May 15, 2022

开始Hugo+Github Pages

参考：https://www.gohugo.org/#/

准备工作准备

安装Hugo，git过程略

配置Hugo路径到环境变量Path

过程略

生成站点

1hugo new site /D:/hugo/huangxdBlog

生成站点到本地路径：/D:/hugo/huangxdBlog

可以看到/D:/hugo/huangxdBlog下已生成初始站点结构：

1  ▸ archetypes/
2  ▸ content/
3  ▸ layouts/
4  ▸ static/
5    config.toml

创建文章

1hugo new about.md

生成路径自动为：/content/about.md

1hugo new post/first.md

生成路径自动为：/content/post/about.md

安装皮肤模板

从 https://www.gohugo.org/theme/#/ 或者 https://themes.gohugo.io/#/挑选合适的模板，下载

1$ cd themes
2$ git clone https://github.com/spf13/hyde.git

本地调试

在本地运行并预览网站效果

1hugo server --theme=angels-ladder --buildDrafts

注意：将hyde换成自己的主题名

如果已经在config.toml中配置了theme，则运行语句可以缩减为：

1hugo server --buildDrafts

浏览器里打开链接，可浏览本地效果:

1 http://localhost:1313

部署

GitHub上先新建一个仓库（命名：#pipony#(换成自己的github用户名).github.io）

生成最终页面

在根目录执行：

1hugo --theme=hyde --baseUrl="http://pipony.github.io/"

同理：在配置文件中若已经指定了主题，则可以忽略--theme参数：

1hugo --baseUrl="http://pipony.github.io/"

此时会在/public目录下生成所有静态页面，再将/public/*上传到github的main分支即可（注意：不要变成master分支了）

（注意，以上命令并不会生成草稿页面，如果未生成任何文章，请去掉文章头部的 draft=true 再重新生成。）

配置

根目录config.toml中，修改baseURL = ‘https://pipony.github.io/' （自己的github地址）

查看效果

1http://pipony.github.io/
